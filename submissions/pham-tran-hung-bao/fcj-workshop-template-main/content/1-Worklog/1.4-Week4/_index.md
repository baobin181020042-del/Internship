**Timeline:** 11/5 → 15/5 (5 working days)

## Day 1 - 11/5: Analyzing EC2 Resources

**Work completed:** I practiced using AWS Compute Optimizer to review recommendations for Amazon EC2 instances. I analyzed CPU and memory utilization, compared the current instance configuration with the recommended configuration, and learned how AWS evaluates resource usage to improve performance and reduce unnecessary costs.

**Result and lesson:** I learned how to interpret AWS Compute Optimizer recommendations and identify when an EC2 instance should be upgraded or downsized based on workload requirements.

## Day 2 - 12/5: Evaluating EC2 Performance

**Work completed:** I continued working with AWS Compute Optimizer by reviewing additional EC2 recommendations and comparing resource utilization across different instance types. I examined performance metrics and identified opportunities to improve resource efficiency.

**Result and lesson:**I gained a better understanding of how performance metrics can be used to optimize EC2 resources while maintaining application stability and reducing operational costs.

## Day 3 - 13/5: Learning AWS Cost Explorer

**Work completed:** I studied AWS Cost Explorer to monitor AWS service costs and usage. I learned how to generate cost reports for different time periods, group expenses by service, and analyze spending trends to identify the main factors contributing to increased AWS costs.

**Result and lesson:** I learned how to monitor and analyze AWS costs effectively, providing a foundation for making informed decisions about cloud budget optimization.

## Day 4 - 14/5:  Optimizing Performance and Costs

**Work completed:** Based on information collected from AWS Compute Optimizer and AWS Cost Explorer, I evaluated the resource utilization of EC2 instances and proposed configuration adjustments to balance application performance with operating costs. I also reviewed best practices for optimizing cloud resources.

**Result and lesson:** I understood the importance of continuously monitoring both system performance and cloud expenses in order to maximize the efficiency of AWS resources.

## Day 5 - 15/5: Weekly Documentation

**Work completed:** I summarized the knowledge gained throughout the week on AWS Compute Optimizer and AWS Cost Explorer. I organized my notes on performance analysis, cost monitoring, and optimization strategies, and documented the practical experience gained during the exercises for my internship report.

**Result and lesson:** By the end of the fourth week, I had developed the ability to analyze AWS resource performance, monitor cloud spending, and recommend practical optimization strategies to improve system efficiency while controlling operational costs.
