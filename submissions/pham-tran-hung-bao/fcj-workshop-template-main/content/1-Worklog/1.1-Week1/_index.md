**Timeline:** 20/4 → 24/4 (5 working days)

## Day 1 - 20/4:  Getting Started with AWS Management Console

**Work completed:** I familiarized myself with the AWS Management Console and explored its interface to understand how AWS services are organized and managed. I reviewed the AWS account settings, selected the Singapore Region (ap-southeast-1), and learned how to navigate through different service dashboards. I also became familiar with the Billing Dashboard and AWS Free Tier to better understand cost management and avoid unnecessary charges during the internship.

**Result and lesson:** After completing this session, I understood the basic structure of the AWS Management Console and learned how to manage AWS resources within the appropriate region. This provided a solid foundation for working with AWS services throughout the internship.

## Day 2 - 21/4: Learning Amazon EC2

**Work completed:** I studied Amazon Elastic Compute Cloud (EC2), focusing on the concept of virtual servers in the cloud. I learned about key EC2 components, including Amazon Machine Images (AMIs), Instance Types, Key Pairs, and Security Groups. In addition, I explored the EC2 instance creation process and understood the basic configuration options required when launching a virtual machine.

**Result and lesson:** Through this learning session, I gained an understanding of how cloud-based virtual servers are deployed and managed on AWS. I also recognized the importance of selecting appropriate instance configurations to balance performance, scalability, and operational cost.

## Day 3 - 22/4: S3 and storage basics

**Work completed:** I continued working with Amazon EC2 by connecting to an EC2 instance using SSH and the previously generated Key Pair. I configured Security Groups to allow secure remote access and investigated common connection issues such as incorrect key files, blocked ports, and misconfigured security rules. Furthermore, I performed basic Linux system administration tasks, including updating packages and monitoring CPU, memory, and disk usage.

**Result and lesson:** This practical session helped me understand the remote management process of EC2 instances and how to troubleshoot common connectivity issues. These skills provide an essential foundation for deploying and managing cloud servers in future projects.

## Day 4 - 23/4: Learning Amazon S3

**Work completed:** I studied Amazon Simple Storage Service (Amazon S3), focusing on concepts such as Buckets, Objects, Storage Classes, and Versioning. I practiced creating S3 buckets, uploading files, organizing stored objects, and configuring access permissions. Additionally, I learned about the Block Public Access feature and other security mechanisms designed to protect stored data from unauthorized access.

**Result and lesson:** After completing this session, I understood how Amazon S3 provides scalable object storage services on AWS. I also recognized the importance of configuring access permissions correctly to maintain data security and prevent accidental exposure of sensitive information.

## Day 5 - 24/4: Weekly Review and Knowledge Consolidation

**Work completed:** I reviewed all the topics covered during the first week, including the AWS Management Console, Amazon EC2, and Amazon S3. I organized my study notes, summarized the hands-on exercises I had completed, and documented important concepts for future reference. Finally, I reviewed and removed unused AWS resources created during practice to avoid unnecessary costs.

**Result and lesson:** By the end of the first week, I had developed a general understanding of the AWS platform and gained practical experience with its core services, particularly Amazon EC2 and Amazon S3. These fundamental skills established a strong foundation for learning networking, database, and security services in the following weeks of the internship.
