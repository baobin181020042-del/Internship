**Mốc thời gian:** 15/6 → 19/6 (5 ngày làm việc)

## Ngày 1 - 15/6: Tìm hiểu AWS Systems Manager

**Công việc đã thực hiện:** Tôi nghiên cứu dịch vụ AWS Systems Manager (SSM), tìm hiểu vai trò của dịch vụ trong việc quản lý và vận hành các EC2 Instance. Đồng thời làm quen với các tính năng chính như Session Manager, Run Command và khả năng quản trị máy chủ từ xa mà không cần sử dụng SSH.

**Kết quả và bài học:** Tôi nắm được cách kết nối và quản trị EC2 bằng Session Manager, góp phần nâng cao tính bảo mật cho hệ thống.

## Ngày 2 - 16/6: Thực hành Session Manager

**Công việc đã thực hiện:** Tôi thực hành sử dụng Session Manager để kết nối đến EC2 Instance thông qua AWS Management Console mà không cần mở cổng SSH. Đồng thời tìm hiểu các điều kiện cần thiết để EC2 có thể sử dụng Session Manager.

**Kết quả và bài học:** Tôi nắm được cách kết nối và quản trị EC2 bằng Session Manager, góp phần nâng cao tính bảo mật cho hệ thống.

## Ngày 3 - 17/6: Thực hành Run Command

**Công việc đã thực hiện:** Tôi tìm hiểu tính năng Run Command và thực hành gửi các lệnh quản trị đến EC2 Instance từ AWS Systems Manager. Đồng thời kiểm tra kết quả thực thi và theo dõi trạng thái của từng lệnh.

**Kết quả và bài học:** Tôi hiểu được cách quản lý nhiều máy chủ cùng lúc thông qua Run Command và giảm bớt thao tác đăng nhập trực tiếp vào từng EC2.

## Ngày 4 - 18/6:  Quản lý EC2 với Systems Manager

**Công việc đã thực hiện:**Tôi tiếp tục thực hành các tính năng của AWS Systems Manager, tìm hiểu cách theo dõi trạng thái EC2, quản lý cấu hình và hỗ trợ thực hiện các tác vụ quản trị từ xa thông qua giao diện của AWS.

**Kết quả và bài học:** Tôi hiểu được lợi ích của AWS Systems Manager trong việc quản lý tập trung và nâng cao hiệu quả vận hành hệ thống.

## Ngày 5 - 19/6: Tổng kết chuẩn bị dự án

**Công việc đã thực hiện:** Tôi tổng hợp lại các kiến thức đã học về AWS Systems Manager, Session Manager và Run Command. Đồng thời rà soát các nội dung đã thực hành, hoàn thiện ghi chú và hệ thống lại các bước quản lý EC2 phục vụ cho báo cáo thực tập.

**Kết quả và bài học:** Kết thúc tuần thứ chín, tôi đã biết cách sử dụng AWS Systems Manager để quản lý EC2 từ xa, thực hiện các tác vụ quản trị và nâng cao tính bảo mật trong quá trình vận hành hệ thống.
