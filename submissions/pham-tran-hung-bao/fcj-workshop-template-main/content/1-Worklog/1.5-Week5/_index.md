**Timeline:** 18/5 → 22/5 (5 working days)

## Day 1 - 18/5: Learning AWS IAM

**Work completed:** I studied AWS Identity and Access Management (IAM) and learned about its role in managing identities and access permissions within AWS. I explored the core IAM components, including Users, Groups, Roles, and Policies, to understand how access permissions are assigned to users and AWS services.

**Result and lesson:** I gained a clear understanding of how IAM manages authentication and authorization in AWS and learned the relationship between Users, Groups, Roles, and Policies.

## Day 2 - 19/5: Creating IAM Users and Groups

**Work completed:** I practiced creating IAM Users and Groups, configuring sign-in credentials, assigning users to groups, and granting permissions using AWS managed policies. I also tested the access permissions of different user accounts after applying the policies.

**Result and lesson:** I learned how to create and manage IAM users efficiently and understood the advantages of assigning permissions through Groups instead of configuring each User individually.

## Day 3 - 20/5: Learning IAM Roles

**Work completed:** I studied IAM Roles and practiced creating roles that allow Amazon EC2 instances to access AWS services without using access keys. I also explored the AssumeRole mechanism and learned how IAM Roles provide secure communication between AWS services.

**Result and lesson:** I understood the differences between IAM Users and IAM Roles and learned how IAM Roles improve security by eliminating the need for long-term access credentials.

## Day 4 - 21/5: Creating IAM Policies

**Work completed:** I practiced creating IAM Policies using the JSON policy language, granting permissions to specific AWS resources while following the Principle of Least Privilege. I also reviewed how policies are attached to Users, Groups, and Roles to control access.

**Result and lesson:** I learned how to create IAM policies that meet different access requirements and understood how the Principle of Least Privilege helps reduce security risks.

## Day 5 - 22/5: Weekly Documentation

**Work completed:** I reviewed the topics covered during the week, including IAM Users, Groups, Roles, and IAM Policies. I verified my configurations, documented important notes from the practice sessions, and organized my learning materials for my internship report.

**Result and lesson:** By the end of the fifth week, I had developed a solid understanding of identity and access management on AWS, including how to manage permissions securely and apply the Principle of Least Privilege when administering cloud resources.
